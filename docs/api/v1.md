# API V1

## `POST /api/v1/sign-up`

Creates a new user account with an email and password.

### Request

- Authentication: none
- Content-Type: `application/json`

Schema:

```json
{
  "email": "string",
  "password": "string"
}
```

Example:

```json
{
  "email": "user@example.com",
  "password": "super-secret"
}
```

### Response

Success:

- Status: `201 Created`

Schema:

```json
{
  "id": 1,
  "email": "string",
  "token": "string"
}
```

Conflict:

- Status: `409 Conflict`

Schema:

```json
{
  "detail": "User with this email already exists"
}
```

## `POST /api/v1/sign-in`

Verifies user credentials and returns the matching user record when authentication succeeds.

### Request

- Authentication: none
- Content-Type: `application/json`

Schema:

```json
{
  "email": "string",
  "password": "string"
}
```

Example:

```json
{
  "email": "user@example.com",
  "password": "super-secret"
}
```

### Response

Success:

- Status: `200 OK`

Schema:

```json
{
  "id": 1,
  "email": "string",
  "token": "string"
}
```

Unauthorized:

- Status: `401 Unauthorized`

Schema:

```json
{
  "detail": "Invalid email or password"
}
```

## `POST /api/v1/get-report-urls`

Returns a download URL for the most recent cached supported SEC report PDF for each requested company.

### Request

- Authentication: bearer token required
- Content-Type: `application/json`

Schema:

```json
{
  "report_type": "10-K",
  "companies": ["Apple", "Meta"]
}
```

Supported report types:

- `10-K`

Allowed company names:

- `Apple`
- `Meta`
- `Alphabet`
- `Amazon`
- `Netflix`
- `Goldman Sachs`

These companies are seeded into the `companies` table and are treated as the supported company registry for the API.
This endpoint does not call SEC directly. It only returns URLs for reports that have already been prefetched and stored by the background workers.

### Response

Success:

- Status: `200 OK`

Schema:

```json
{
  "Apple": "string",
  "Meta": "string"
}
```

Validation error:

- Status: `422 Unprocessable Entity`

Schema:

```json
{
  "detail": "string"
}
```

Example invalid report type response:

```json
{
  "detail": "Invalid value of report type. Supported report types: ['10-K']"
}
```

Example invalid company names response:

```json
{
  "detail": "Invalid value of company name. Supported companies names: ['Alphabet', 'Amazon', 'Apple', 'Goldman Sachs', 'Meta', 'Netflix']"
}
```

Report not available yet:

- Status: `400 Bad Request`

Schema:

```json
{
  "detail": "Report is not available yet for company: Apple"
}
```

Missing authentication:

- Status: `403 Forbidden`

Invalid or expired token:

- Status: `401 Unauthorized`

Schema:

```json
{
  "detail": "Invalid or expired token"
}
```

## `GET /api/v1/files/{file_name}`

Returns a stored PDF report file for an authenticated user.

This endpoint is used by the URLs returned from `POST /api/v1/get-report-urls` when local storage is active.

### Request

- Authentication: bearer token required
- Path parameter:
  - `file_name`: stored PDF filename returned by the report generation flow

### Response

Success with local storage:

- Status: `200 OK`
- Content-Type: `application/pdf`

Success with S3 storage:

- Status: `307 Temporary Redirect`
- Location header points to the object URL

Not found:

- Status: `404 Not Found`

Schema:

```json
{
  "detail": "File not found"
}
```

Missing authentication:

- Status: `403 Forbidden`

Invalid or expired token:

- Status: `401 Unauthorized`

Schema:

```json
{
  "detail": "Invalid or expired token"
}
```
